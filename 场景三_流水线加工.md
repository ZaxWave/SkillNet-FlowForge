# 场景三：流水线式设备加工（阶段并行 + 共享设备互斥）

**总任务：** "找到土豆，洗干净，用微波炉加热，然后放到餐桌上；同时找到苹果，洗干净，放到冰箱里冷却。"

---

## 场景特点

两条加工流水线——热加工 vs 冷加工，中间共享水槽。展示"大部分并行 + 局部互斥"的模式。

---

## 第一阶段：解析 + 扫描（串行）

```
alfworld-goal-interpreter → 拆为 2 条独立加工链:
  加工链1: 土豆 → 洗 → 加热 → 餐桌
  加工链2: 苹果 → 洗 → 冷却 → (冰箱)

alfworld-environment-scanner → 定位:
  物品: 土豆（台面）、苹果（果篮）
  设备: 水槽、微波炉、冰箱、餐桌
```

---

## 第二阶段：两路并行加工链

```
【分支A】土豆 → 热加工链               【分支B】苹果 → 冷加工链
=========================              =========================
object-locator(土豆)                    object-locator(苹果)
    │                                      │
object-picker                           object-picker
    │                                      │
appliance-navigator(sinkbasin)          appliance-navigator(sinkbasin)
    │                                      │
clean-object                            clean-object
    │                                      │
appliance-navigator(microwave)          appliance-navigator(fridge)
    │                                      │
appliance-preparer(开微波炉门)          appliance-preparer(开冰箱门)
    │                                      │
object-heater                           object-cooler
    │                                      │
location-navigator(diningtable)         object-placer(留在冰箱或指定位置)
    │                                      │
object-placer                           receptacle-closer(关冰箱)
    │                                      │
task-verifier                           task-verifier
```

---

## 关键问题：共享水槽

两条链都需要 `clean-object` → 都用到 `sinkbasin`。如果只有一个水槽，必须互斥排队。

```
时间线:
═════════════════════════════════════════════════════════

分支A: ──定位──拿取──前往水槽──[清洗]──前往微波炉──加热──放置──验证──
分支B: ──定位──拿取──────────排队────[清洗]──前往冰箱──冷却──放置──验证──
                               ↑
                          水槽互斥区 (同时只一个使用)

═════════════════════════════════════════════════════════
```

互斥策略：
1. 先到达的先占用
2. 占用者用完执行 `receptacle-closer` 释放
3. 排队者获知水槽空闲后继续

---

## 技能复用的效率

两条链大量复用同一组 skill，仅加热/冷却阶段不同：

| 阶段 | 分支A | 分支B | 复用 |
|---|---|---|---|
| 定位 | object-locator | object-locator | ✅ |
| 拿取 | object-picker | object-picker | ✅ |
| 清洗 | clean-object | clean-object | ✅（互斥） |
| 加工 | object-heater | object-cooler | ❌ 不同 |
| 放置 | object-placer | object-placer | ✅ |
| 验证 | task-verifier | task-verifier | ✅ |

---

## 第三阶段：汇总验证

```
综合 task-verifier:
  ✅ 餐桌: 1 热土豆
  ✅ 冰箱: 1 冷苹果 / 已冷苹果在指定位置
```

---

## 并行度

| 阶段 | 并行数 | 说明 |
|---|---|---|
| 解析 | 1 | 拆目标 |
| 定位+拿取 | **2** | 各自找，互不干扰 |
| 水槽清洗 | 1 | 共享设备，互斥 |
| 加工 | **2** | 微波炉/冰箱不同设备，可并行 |
| 放置 | **2** | 不同终点 |
| 验证 | 1 | 汇总 |

---

## 拓展变体：三链并行 + 两共享设备

```
土豆 → 洗 → 加热 → 餐桌
苹果 → 洗 → 冷却 → 冰箱
杯子 → 洗 → 放回 → 杯架

三条链并行，都竞争水槽 → 需要排队调度器
最大并行度 = 3（定位/加工/放置阶段）
互斥退化为 1（清洗阶段）
```
