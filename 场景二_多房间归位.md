# 场景二：多房间物品归位（空间并行）

**总任务：** "把客厅的遥控器放到茶几上，把厨房的刀放到抽屉里，把卧室的手机放到床头柜上。"

---

## 场景特点

三个搬运子任务分布在三个不同房间，物品、起点、终点全部独立，**零资源冲突**。这是并行度最干净的场景。

---

## 第一阶段：解析 + 全屋扫描（串行）

```
alfworld-goal-interpreter → 拆出 3 个独立搬运子任务:
  子任务1: 遥控器 → 茶几
  子任务2: 刀 → 抽屉
  子任务3: 手机 → 床头柜

alfworld-environment-scanner → 标注各房间物品分布:
  客厅: [遥控器, 台灯, 抱枕, ...]
  厨房: [刀, 砧板, 锅, ...]
  卧室: [手机, 枕头, ...]
```

---

## 第二阶段：三路并行

```
          ┌──────────────┬──────────────┬──────────────┐
          ▼              ▼              ▼
      【分支A】       【分支B】       【分支C】
      客厅-遥控器    厨房-刀        卧室-手机
       → 茶几        → 抽屉         → 床头柜
```

### 分支A — 客厅：遥控器 → 茶几

```
object-locator(遥控器) → receptacle-opener(如果需要)
→ object-picker → location-navigator(茶几)
→ receptacle-preparer → object-placer
→ task-verifier
```

### 分支B — 厨房：刀 → 抽屉

```
object-locator(刀) → receptacle-opener
→ object-picker → location-navigator(抽屉)
→ receptacle-preparer → object-storer
→ task-verifier
```

> 用 `object-storer` 而非 `object-placer`，因为抽屉需要先打开、放入后关闭。

### 分支C — 卧室：手机 → 床头柜

```
object-locator(手机) → receptacle-opener
→ object-picker → location-navigator(床头柜)
→ receptacle-preparer → object-placer
→ task-verifier
```

---

## 第三阶段：汇总验证（并行检查）

三个分支各自带 `task-verifier`，验证互不依赖，可以一并完成：

```
✅ 客厅茶几: 有遥控器
✅ 厨房抽屉: 有刀
✅ 卧室床头柜: 有手机
→ 全部通过 = 任务完成
```

---

## 并行度

| 阶段 | 并行数 | 说明 |
|---|---|---|
| 解析 | 1 | 拆目标 |
| 执行 | **3** | 三个房间完全独立 |
| 验证 | **3** | 各自独立验证 |

---

## 技能链对比

| 分支 | 技能数 | 核心差异 |
|---|---|---|
| A（遥控器） | 7 | 纯搬运，object-placer |
| B（刀） | 7 | 存抽屉，object-storer |
| C（手机） | 7 | 纯搬运，object-placer |

三条链结构几乎一致，区别在终点类型（开敞容器 vs 封闭容器）。

---

## 拓展变体

如果加入更多房间和物品，并行度可以线性扩展：

```
客厅: 遥控器→茶几, 杂志→书架, 杯子→杯垫      (3子任务)
厨房: 刀→抽屉, 盐→调味架, 锅盖→挂钩           (3子任务)
卧室: 手机→床头柜, 衣服→衣柜, 书→书桌          (3子任务)
卫生间: 毛巾→架子, 牙刷→杯子里                 (2子任务)

最大并行度 = 11
```
