# 场景四：批量收集（数量驱动并行）

**总任务：** "找到两支铅笔和两本书，全部放到书桌上。"

---

## 场景特点

核心是 `inventory-management` 驱动循环收集。两个物品类型互不依赖，各自独立循环。展示"循环内串行、循环间并行"的模式。

---

## 第一阶段：解析 + 扫描（串行）

```
alfworld-goal-interpreter → 结构化目标:
  {
    铅笔: { 需要: 2, 去向: 书桌, 优先搜: [抽屉, 笔筒, 桌子] }
    书:   { 需要: 2, 去向: 书桌, 优先搜: [书架, 桌子, 床头柜] }
  }

alfworld-environment-scanner → 列出所有可能容器及其内容物
```

---

## 第二阶段：两路并行收集

```
【分支A】铅笔×2                             【分支B】书×2
=================                          ================
inventory-management(铅笔, 2)               inventory-management(书, 2)
    │                                          │
  ┌─[循环]──┐                               ┌─[循环]──┐
  │          │                               │          │
  │ object-locator(铅笔)                    │ object-locator(书)
  │   优先搜: 抽屉→笔筒→桌子                 │   优先搜: 书架→桌子→床头柜
  │          │                               │          │
  │ search-pattern-executor                 │ search-pattern-executor
  │   (候选容器列表)                         │   (候选容器列表)
  │          │                               │          │
  │ object-retriever(拿取)                  │ object-retriever(拿取)
  │          │                               │          │
  │ location-navigator(书桌)                │ location-navigator(书桌)
  │          │                               │          │
  │ object-placer(放置)                     │ object-placer(放置)
  │          │                               │          │
  │ inventory-management:                   │ inventory-management:
  │   collected += 1                        │   collected += 1
  │          │                               │          │
  └─[collected < 2 ? 继续 : 退出]──┘        └─[同左]──┘
    │                                          │
task-verifier(确认2支铅笔在书桌)            task-verifier(确认2本书在书桌)
```

---

## 循环展开示意

```
分支A (铅笔):
  第1轮: 发现抽屉1有2支铅笔 → 拿1支 → 放书桌 → count=1
  第2轮: 仍去抽屉1 → 拿第2支 → 放书桌 → count=2 → 完成

分支B (书):
  第1轮: 搜到书架有3本书 → 拿1本 → 放书桌 → count=1
  第2轮: 书架上还有 → 再拿1本 → 放书桌 → count=2 → 完成
```

两分支同时跑，互不干扰。

---

## 第三阶段：合并验证

```
综合 task-verifier:
  书桌上:
    pencil 1 ✅
    pencil 2 ✅
    book 1   ✅
    book 2   ✅
  → 任务完成
```

---

## 资源冲突分析

| 情况 | 冲突? | 处理 |
|---|---|---|
| 铅笔和书在不同容器 | 无 | 完全并行 |
| 铅笔和书都在抽屉1 | 有 | 排队取——先到先取，后者等容器空闲 |
| 第1轮没找够数量 | — | `search-verifier` 回溯已访问容器再搜 |
| 书桌上已有物品 | 无 | 追加放置，不冲突 |

---

## 并行度

| 阶段 | 并行数 | 说明 |
|---|---|---|
| 解析 | 1 | 拆目标 |
| 收集 | **2** | 铅笔和书各自独立循环 |
| 单分支内循环 | 1 | 每轮拿→送→回，串行 |
| 验证 | 1 | 汇总 |

---

## 对比：不用 inventory-management 的朴素写法

```
朴素: object-locator → take 2 pencils → go desk → put 2 → go back → find 2 books → take → go desk → put
问题: 一次只能拿一件物品，拿两件需要中间放下 → 多一轮来回

优化后 (inventory-management):
  每拿一件立刻送书桌 → 运输路径更短 → 并行时两个循环交替跑，不互相阻塞
```

---

## 拓展变体

### 变体A：多类型 + 多数量

```
任务: "收集 3 支笔、2 个夹子、1 把剪刀，全放文具盒"
→ 3 路并行，每路有各自的 inventory-management 循环
→ 最大并行度 = 3
```

### 变体B：收集 + 加工 + 放置

```
任务: "找 2 个杯子，洗干净，放托盘"
→ 循环体升级: object-locator → pick → clean(sinkbasin) → placer(tray)
→ 每轮多了一个加工步骤，但循环结构不变
```
